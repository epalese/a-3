Here is a brief explanation on how to use the logging levels in REDS.

SEVERE: use it for severe exceptions that my break correct operation. Print
	the stack trace of the exception on System.err 

WARNING: use it for exceptions that was possible to manage in a reasonable
	 way. Do not print the stack trace, but start the message with the exception
	 (e.g., logger.warning(ex+" doing...");

INFO: use it to signal important events (e.g. adding/removing neighbors/links)

CONFIG: use it to notify about software configuration (e.g., adding a new
	transport to an overlay).

FINE: use it to trace the execution of the system

FINER: use it for further tracing

FINEST: do not use
